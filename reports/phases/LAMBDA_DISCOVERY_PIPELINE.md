# LAMBDA_DISCOVERY_PIPELINE

Orchestrates the end-to-end discovery process from hypothesis to knowledge update. Ensures traceability and evidence-based progression.

## Pipeline Stages

### 1. Hypothesis
- Generated by Hypothesis Generation module
- Includes trigger provenance and initial evidence

### 2. Experiment
- Submitted to Experiment Engine for validation
- Multiple experimental methods applied

### 3. Evidence
- All experimental results compiled
- Confidence scores calculated
- Strength of evidence assessed

### 4. Decision
- Governance module reviews evidence and confidence
- Approves or rejects knowledge update
- May request additional experiments

### 5. Graph Update
- Approved updates committed to knowledge graph
- Rejected hypotheses recorded as lessons
- Related hypotheses re-evaluated

## Traceability

Every pipeline instance is tracked with:
- Unique pipeline ID
- All hypothesis IDs processed
- All experiment IDs run
- Governance decision and reasoning
- Timestamps for each stage

## Commit Criteria

Updates only commit when:
- Evidence count >= min_evidence (typically 2+ independent experiments)
- Confidence >= min_confidence (typically 0.5)
- No conflicting high-confidence knowledge

## Lessons Repository

Rejected hypotheses and failed experiments are archived as lessons. These contribute to:
- Future hypothesis generation (avoiding dead ends)
- Self-assessment metrics (learning from failures)
- Audit trail for transparency

## Rollback Support

If later evidence contradicts a committed discovery, the pipeline supports rollback through the autonomy governance's rollback stack.
